1. Introdução e Objetivos

O MedAgenda é uma aplicação web desenvolvida para o agendamento de consultas médicas. O sistema permite que pacientes se registem, escolham um médico, agendem consultas e acompanhem o estado das suas marcações.

Por outro lado, os profissionais de saúde têm acesso a um painel administrativo privado onde podem visualizar as suas consultas, ver os dados dos pacientes e atualizar o estado das consultas (concluída, cancelada, reagendar).

Objetivos

  • Simplificar o processo de agendamento de consultas médicas
  • Providenciar uma plataforma acessível via browser e instalável como aplicação móvel
  • Separação clara entre a interface pública (pacientes) e o painel administrativo (médicos)
  • Funcionamento offline através de tecnologias PWA
  • Design responsivo e intuitivo para todos os dispositivos

4.5 Storage

O Supabase Storage é utilizado para armazenar as fotos de perfil dos médicos. É criado um bucket público chamado fotos-medicos com políticas de acesso que permitem upload, download, atualização e eliminação.

5. Funcionalidades — Paciente

5.1 Registo e Login

O paciente pode criar uma conta fornecendo o seu nome, email, telefone e password. O sistema utiliza a autenticação do Supabase, que cria automaticamente o perfil na tabela utilizadores através de um trigger.

5.2 Agendamento de Consultas

Na página de agendamento, o paciente pode:

  • Selecionar um médico da lista (carregada da base de dados)
  • Escolher a data da consulta (a partir do dia atual)
  • Selecionar um horário disponível (intervalos de 30 minutos, das 08:00 às 17:00)
  • Escolher o tipo de consulta (Geral, Especialidade, Urgência, Exame)
  • Adicionar notas ou observações opcionais

5.3 Painel do Paciente (Dashboard)

O dashboard apresenta:

  • 3 cards de estatísticas: Consultas Agendadas, Concluídas e Canceladas
  • Tabela com todas as consultas, com filtros (Todas, Agendadas, Canceladas)
  • Ações: Editar consulta e Cancelar consulta

5.4 Perfil do Paciente

O paciente pode editar o seu nome e telefone. O email não é editável pois é utilizado para autenticação.

6. Funcionalidades — Médico (Painel Admin)

O painel dos médicos é uma área privada, acessível através de um URL específico (/medico.html) e de um botão discreto no canto inferior direito do site público.

6.1 Login

O médico seleciona o seu nome numa lista dropdown e insere a sua senha de acesso (por defeito: 1234). A autenticação é feita diretamente contra a tabela medicos, sem utilizar o sistema de autenticação do Supabase.

6.2 Dashboard do Médico

O painel apresenta:

  • Estatísticas: Consultas Hoje, Próximas, Total
  • As Minhas Consultas: Lista em cards com dados completos do paciente:
    • Nome do paciente
    • Email
    • Telefone
    • Data e hora
    • Tipo de consulta
    • Estado atual
    • Notas/observações
  • Filtros: Todas, Hoje, Agendadas, Concluídas, Canceladas
  • Ações:
    • Responder: Abre modal para atualizar estado (Concluída, Cancelada, Reagendar) e adicionar notas
    • Cancelar: Cancelamento rápido de consulta

6.3 Edição de Perfil

O médico pode:

  • Alterar o seu nome
  • Fazer upload de uma foto de perfil (guardada no Supabase Storage)
  • Remover a foto de perfil
  • Alterar a senha de acesso

7. Progressive Web App (PWA)

O MedAgenda foi convertido numa Progressive Web App, permitindo que seja instalado como uma aplicação nativa em dispositivos Android e iOS.

7.1 Componentes PWA

ComponenteFicheiroFunção
Web App Manifestmanifest.jsonDefine nome, ícones, cores e comportamento da app
Service Workersw.jsCache de recursos para funcionamento offline
Install Bannerjs/install.jsBanner automático de instalação
Íconesicons/Ícones da app (192x192 e 512x512)

7.2 Estratégia de Cache

O Service Worker utiliza a estratégia Cache First: todos os recursos estáticos (HTML, CSS, JS) são guardados em cache na primeira visita. Nas visitas seguintes, os recursos são servidos do cache, garantindo funcionamento offline. Pedidos à API do Supabase são sempre feitos diretamente à rede.

7.3 Instalação

Quando o utilizador acede ao site no navegador móvel, um banner aparece automaticamente após 2 segundos oferecendo a opção de instalar a aplicação. Após a instalação, o MedAgenda aparece como um ícone no ecrã inicial, abrindo em modo standalone (sem barra de endereço do navegador).

8. Segurança

8.1 Autenticação

  • Pacientes: Autenticação via Supabase Auth (email + password com hash seguro)
  • Médicos: Autenticação simples (nome + senha contra a tabela medicos)

8.2 Row Level Security (RLS)

  • utilizadores: RLS desativado (médicos precisam de aceder aos dados dos pacientes)
  • consultas: RLS desativado para leitura (médicos veem todas as suas consultas); escrita restrita ao utilizador autenticado
  • medicos: RLS desativado (acesso público à lista para seleção no agendamento)

8.3 Supabase Storage

O bucket fotos-medicos tem políticas que permitem operações de leitura, escrita, atualização e eliminação. As fotos são públicas para visualização.

9. Design e Interface

9.1 Tema

O sistema utiliza um tema escuro profissional com as seguintes características:

  • Fundo: Tons de preto/cinzento escuro (#09090b a #18181b)
  • Cor principal: Roxo/Indigo (#6366f1)
  • Texto: Branco (#fafafa) com variações de cinzento para texto secundário
  • Badges de estado: Azul (agendada), Verde (concluída), Vermelho (cancelada)

9.2 Responsividade

O layout é totalmente responsivo, adaptando-se a ecrãs desde telemóveis (320px) até desktops (1920px+). São utilizados breakpoints do Bootstrap 5 para reorganizar os elementos conforme o tamanho do ecrã.

9.3 Componentes UI

  • Cards para estatísticas e consultas
  • Tabelas responsivas com filtros
  • Modais para edição e confirmação
  • Badges coloridos para estados
  • Botões de ação com ícones
  • Formulários com validação
  • Alertas de sucesso/erro

10. Conclusão

O MedAgenda é uma solução completa e funcional para o agendamento de consultas médicas. O sistema cumpre os objetivos propostos:

  • Simplicidade: Interface intuitiva que permite agendar uma consulta em poucos cliques
  • Acessibilidade: Disponível em qualquer dispositivo com um browser, e instalável como aplicação móvel
  • Separação de responsabilidades: Interface pública para pacientes e painel administrativo privado para médicos
  • Funcionalidade offline: Graças à implementação PWA, o sistema funciona mesmo sem ligação à internet
  • Tecnologias modernas: Utilização de frameworks e serviços atuais que garantem escalabilidade e manutenção facilitada

O projeto demonstra a viabilidade de criar aplicações web completas utilizando apenas tecnologias do lado do cliente (HTML, CSS, JavaScript) combinadas com serviços Backend-as-a-Service como o Supabase, eliminando a necessidade de servidores próprios.
